# Todo List API

Простое API на **Python + FastAPI** для управления задачами (Todo list) с возможностью добавления, редактирования, удаления задач, а также установки приоритета и дедлайнов.

## Технологии
* **Backend:** Python, FastAPI
* **Database:** PostgreSQL
* **Контейнеризация:** Docker, Docker Compose

---

## Необходимый софт
Для запуска проекта на вашем компьютере должны быть установлены:
* [Docker](https://www.docker.com/)
* [Docker Compose](https://docs.docker.com/compose/)
* Git

---

## Установка и запуск

### Клонирование репозитория
```bash
git clone <ссылка_на_ваш_репозиторий>
cd <название_папки_проекта>
```

### Переменные окружения
Для работы приложения необходимо настроить переменные окружения. Создайте файл `.env` в корне проекта (на одном уровне с `docker-compose.yml`) со следующим содержимым:

```env
POSTGRES_USER=myuser
POSTGRES_PASSWORD=mypassword
POSTGRES_DB=tododb
```
*(При необходимости измените значения под ваше окружение)*

### Запуск
Запустите сборку и старт контейнеров в фоновом режиме:

```bash
docker-compose up --build -d
```

После успешного запуска интерактивная документация (Swagger UI) будет доступна по адресу:
🔗 **http://localhost:8000/docs**

## Модель задачи (Todo)
Каждая задача в базе данных содержит следующие поля:
* `id` – уникальный идентификатор
* `title` – заголовок (обязательное поле)
* `description` – описание (опционально)
* `priority` – приоритет (1 – высокий, 2 – средний, 3 – низкий; по умолчанию 3)
* `due_date` – дедлайн (дата, опционально)
* `completed` – флаг выполнения (по умолчанию `false`)
* `created_at` – дата и время создания
* `updated_at` – дата и время последнего обновления

## API-Эндпоинты

### GET
* **`/api/todos`** — получить список всех задач. *(Сортировка: сначала невыполненные, затем выполненные, внутри по приоритету (1 → 3) и по дедлайну).*
* **`/api/todos/{id}`** — получить информацию о конкретной задаче по её ID.

### POST
* **`/api/todos`** — создать новую задачу.

*Пример тела запроса:*
```json
{
  "title": "Подготовить отчёт",
  "description": "Собрать данные по задачам",
  "priority": 1,
  "due_date": "2026-03-10"
}
```

### PUT
* **`/api/todos/{id}`** — обновить задачу (можно менять заголовок, описание, приоритет, срок и статус).

*Пример тела запроса:*
```json
{
  "title": "Подготовить итоговый отчёт",
  "description": "Обновлённое описание",
  "priority": 2,
  "due_date": "2026-03-12",
  "completed": true
}
```

### DELETE
* **`/api/todos/{id}`** — удалить задачу.

### Прочее
* **`/health`** — простой эндпоинт проверки работоспособности сервиса.

*Ожидаемый ответ:*
```json
{
  "status": "ok"
}
```
